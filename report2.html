Building a Cloud Storage Web App on Telegram

Telegram offers effectively unlimited free cloud storage for user files (up to 2 GB per file on normal accounts, 4 GB on Premium)[1][2]. Several open-source projects exploit this by using Telegram’s API as a “backend”. For example, UnlimCloud is a desktop app that uses your Telegram account as a private storage identifier[3][4], and Pentaract/TeleSync are tools that upload files to a personal Telegram channel. Using Telegram lets you store photos, videos and documents on Telegram’s servers for free[1].

To build your own web application (a “Google Drive–like” site) with Telegram as storage, key ideas are: each user’s files are kept in a dedicated Telegram channel or chat, and your server (via bots or the Telegram Client API) uploads and downloads files in that channel. User files are referenced by Telegram file IDs in your database, so users can list, share, and retrieve them through your website. The general architecture is: a web frontend (HTML/JS) for uploads/downloads, a backend server (e.g. Node.js, Python, Go) to handle the UI and database, and Telegram API integration that moves the files into Telegram.

  • Unlimited Storage via Telegram: Telegram provides “unlimited” storage in practice[1], so you won’t run out of space. The Bot API (and Telegram client API) now supports files up to ~2 GB[2]. (Premium users can push this to 4 GB[5].) You can also split larger files into chunks if needed[6].
  • Channels/Chats per User: A common design (used in Pentaract[7] and the “telegram-cloud” project[8]) is to give each user their own private Telegram channel or group. Your Telegram bot (or bots) is admin in that channel. When the user uploads a file on your site, the backend bot sends that file into the user’s channel and records the resulting file ID. To download, the bot fetches the file from the channel. This isolates users’ files and keeps them private.
  • Telegram API Method: You can use the Telegram Bot API or the full Telegram Client (MTProto) API. Bots are easy to set up (just create a bot via BotFather) but historically had a 50 MB upload limit; however, current Bot API supports ~2 GB files[2]. The Telegram Client API (using libraries like Telethon or Pyrogram) logs in as a normal user and can upload up to 1.5–2 GB[9]. In practice, many builders use a bot account to manage uploads but route large file transfers through a user account API (via Telethon) to bypass smaller limits[9][6].
  • Chunking Large Files: For files above the single-file limit, split into parts. Pentaract demonstrates splitting files into 20 MB chunks (Telegram’s old download limit) and reassembling them[6]. This allows effectively unlimited file sizes.
  • Backend Storage & Metadata: Your server should maintain a database (e.g. SQLite or Postgres) of each user’s file entries: filenames, sizes, timestamps, and Telegram file IDs. This enables listing files in the web UI and retrieving/deleting them. For example, the Telegram-Cloud project keeps a schema.json of all files[10]. You can similarly persist metadata in a database or JSON on your server.
  • Web App UI: Build a standard file-manager interface. Users can drag/drop or use a file picker to upload, see a list/grid of their stored files, and download or delete them. Use any modern web stack (React/Vue or plain JS + HTML) and a backend (Express, Flask, etc). Your site should look polished (like a professional cloud service) and handle errors gracefully. Consider pre-building or using open-source admin panels for file sharing. The UI communicates with your backend via REST or WebSockets.
  • Telegram Login / Authentication: You need a way to authenticate each user to your site. Options: implement a simple username/password system (as in the Telegram-Cloud example[11]), or use Telegram’s OAuth widget so users “login with Telegram”. Either way, once authenticated, your backend knows which Telegram channel/chat belongs to that user.
  • Privacy & Encryption: By default, Telegram stores files on its servers (the cloud is encrypted in transit and at rest, but not end-to-end by your own key). For full privacy, consider encrypting files on the client or server before upload, and decrypting on download (like the Telegram-Cloud project’s FILE_ENCRYPTION option[12][13]). This way even Telegram cannot read the contents. Note encryption inflates file size (e.g. a 17 MB file became 21 MB after encryption in one test[12]), so account for that overhead when splitting chunks.
  • Deployment & Self-Hosting: Host your web app on your own server or VPS. You can run the backend inside Docker (Pentaract suggests a Docker-compose setup[14]) with a database. Ensure HTTPS for security. Since you’ll be using the Telegram Bot API, also secure your bot token on the server. When self-hosted, you control the data flow; users (or you) can expand it to multiple accounts later if you want to offer it as a service.
  • Performance Limits: Be aware of Telegram’s rate limits: bots are limited to ~20 messages per minute by default. This throttles upload/download speed per bot[15]. You may need multiple bot “workers” to parallelize if you have heavy use. Also handle the case where Telegram’s API might be momentarily unavailable.

In summary, to build this “Google Drive on Telegram”:

·       Set up a Telegram bot (via BotFather) and create a private Telegram channel or group for each user’s storage.

·       Develop a web frontend for file upload/download that calls your backend.

·       On upload, have the backend bot send the file (or file chunks) into the user’s channel via the Telegram API. Save the returned file IDs and metadata in your database.

·       On download, fetch the file from Telegram using its file ID and stream it to the user.

·       (Optionally) Encrypt/decrypt files so only your user can read them.

·       Make the UI clean and easy to use (drag-and-drop, progress bars, file previews, etc).

·       Host the service yourself (e.g. on a VPS or private server) so all data flow is controlled by you.

By following these steps and learning from existing open-source projects (like Pentaract[7][6] and the Telegram-Cloud project[8][13]), you can create a professional-looking, private cloud-storage website that uses Telegram’s storage as the backend. This gives you an easy-to-use, self-hosted file storage solution while leveraging Telegram’s free unlimited storage[1][2].

Key References: Projects like UnlimCloud and Pentaract demonstrate the Telegram-based approach[3][6]. They show how a dedicated channel and bot can store unlimited files, splitting large ones into chunks[6]. The official Bot API documentation confirms Telegram supports up to 2 GB per file[2], and community guides note that using the full API (via Telethon) can handle files up to 1.5–2 GB[9]. By integrating these techniques, you can build a user-friendly, secure web app that meets all your requirements.
